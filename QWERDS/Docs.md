## Обзор системы

Проект построен на базе игрового движка, подобного Unity, для MonoGame. Он разделён на две части: **ядро движка** (пространство имён `MyGameEngine`) и **игровую логику** (пространство имён `QWERDS`).

**Ключевые концепции движка:**
- `GameObject` – контейнер для компонентов.
- `Transform` – обязательный компонент, задающий положение, размер, привязку и иерархию.
- `Behaviour` – базовый класс для всех компонентов, содержит методы `Start()` и `Update()`.
- `Scene` – статический менеджер всех объектов, обновления и отрисовки.
- `ResolutionManager` – адаптация координат под любое разрешение с сохранением пропорций.
- `InputManager` – централизованный ввод (клавиатура, мышь, текст).

**Игровая механика (QWERDS):**
- Игрок управляет отрядом роботов, отправляя слова.
- Каждая буква слова запускает действия роботов и врагов (если привязаны).
- Между боями настраивается «протокол» – привязка букв к действиям.
- Слова проверяются по словарю; особые слова дают спецэффекты.

## Базовые классы движка

### `Behaviour`

Абстрактный базовый класс для всех компонентов.

| Член | Описание |
|------|----------|
| `GameObject` | Ссылка на родительский GameObject. |
| `Transform` | Краткий доступ к `GameObject.Transform`. |
| `Enabled` | Если `false`, `Update()` не вызывается. |
| `Start()` | Вызывается однократно перед первым `Update()`. |
| `Update(GameTime gameTime)` | Вызывается каждый кадр. |
| `OnDestroy()` | Вызывается при уничтожении компонента или объекта. |

### `GameObject`

Контейнер компонентов. Всегда содержит `Transform`.

| Член | Описание |
|------|----------|
| `Name` | Имя объекта (для отладки). |
| `Transform` | Ссылка на компонент Transform. |
| `ActiveSelf` | Локальная активность. |
| `ActiveInHierarchy` | Активность с учётом родителей. |
| **Конструктор** | `GameObject(string name, Transform transform, params Behaviour[] components)` |
| `AddComponent<T>()` | Создаёт и добавляет компонент типа `T`. |
| `GetComponent<T>()` | Возвращает первый компонент типа `T` на объекте. |
| `GetComponentInChildren<T>()` | Рекурсивно ищет компонент типа `T` в иерархии. |
| `GetComponents<T>()` | Возвращает все компоненты типа `T`. |
| `Destroy()` | Уничтожает объект и его компоненты, удаляет из сцены. |

> **Важно:** для создания объектов используйте `Scene.CreateGameObject()`.

### `Transform`

Задаёт пространственные характеристики и иерархию. Поддерживает два режима размера по каждой оси: `Fixed` (фиксированный) и `Stretch` (растянутый с отступами). Все координаты задаются в эталонных единицах 1920×1080.

| Свойство / Метод | Описание |
|------------------|----------|
| `Position` | Смещение от точки `Anchor` в эталонных пикселях. |
| `Rotation` | Поворот в радианах. |
| `Anchor` | Нормализованная точка (0..1) на родительском прямоугольнике, к которой привязывается объект. (0,0) – левый верхний угол, (1,1) – правый нижний. |
| `Origin` | Нормализованная точка (0..1) внутри самого объекта; центр поворота и привязки. |
| `Size` | Фиксированный размер (используется, когда `SizeMode` = `Fixed`). |
| `SizeModeX`, `SizeModeY` | Режим размера по осям: `Fixed` или `Stretch`. |
| `StretchLeft`, `StretchRight`, `StretchTop`, `StretchBottom` | Отступы от краёв родителя при `Stretch`. |
| `Parent` | Родительский Transform (или `null`). |
| `Children` | Список дочерних Transform’ов. |
| `SetParent(Transform parent, bool worldPositionStays = true)` | Устанавливает родительский Transform. |
| `GetWorldPosition()` | Мировая позиция (верхний левый угол) в эталонных единицах. |
| `GetWorldSize()` | Мировой размер с учётом режимов `SizeMode`. |
| `GetScreenPosition()` | Позиция в пикселях текущего экранного разрешения. |
| `GetScreenSize()` | Размер в пикселях текущего экранного разрешения. |

### `Scene`

Статический класс-менеджер, управляющий всеми игровыми объектами.

| Метод / Свойство | Описание |
|------------------|----------|
| `Initialize()` | Инициализирует сцену (вызывать до создания объектов). |
| `CreateGameObject(...)` | Создаёт GameObject с заданным Transform и компонентами, добавляет в сцену. |
| `AddGameObject(go)` | Добавляет существующий GameObject. |
| `RemoveGameObject(go)` | Удаляет GameObject из сцены. |
| `LoadContent(ContentManager)` | Загружает текстуры и шрифты для всех SpriteRenderer и UIText, сохраняет `Content` для дозагрузок. |
| `Content` | Статическое свойство – `ContentManager`, доступный после `LoadContent`. |
| `Update(GameTime)` | Обновляет все активные объекты (потокобезопасно). |
| `Draw(SpriteBatch)` | Отрисовывает все SpriteRenderer и UIText с учётом `SortingLayer` и `LayerDepth`. |
| `FindComponentsOfType<T>()` | Возвращает все компоненты указанного типа. |

## Система координат и масштабирование

### ResolutionManager

Обеспечивает отображение эталонной области 1920×1080 на любом экране с центрированием и масштабированием.

| Член | Описание |
|------|----------|
| `ReferenceWidth` (1920) | Эталонная ширина. |
| `ReferenceHeight` (1080) | Эталонная высота. |
| `ActualWidth` / `ActualHeight` | Текущие размеры экрана. |
| `Scale` | Масштабный коэффициент. |
| `Offset` | Смещение в пикселях для центрирования. |
| `Initialize(GraphicsDevice)` | Инициализация (вызывается после создания устройства). |
| `ToScreen(Vector2 refPos)` | Перевод эталонных координат в экранные. |
| `ToScreenSize(Vector2 refSize)` | Перевод эталонного размера в экранные пиксели. |
| `ToReference(Vector2 screenPos)` | Обратное преобразование (например, для мыши). |

## Система ввода

### InputManager

Статический класс, который **один раз за кадр** считывает состояния клавиатуры и мыши. Все компоненты используют его методы, **не вызывая** `Keyboard.GetState()` / `Mouse.GetState()` напрямую.

| Член | Описание |
|------|----------|
| `Update()` | Считывает текущие состояния устройств, сохраняет предыдущие. Должен вызываться **в начале** `Game1.Update()`. |
| `FeedTextInput(char c)` | Добавляет символ от системного события ввода (подписка на `Window.TextInput`). |
| `TryGetTextInput(out char c)` | Возвращает символ из очереди ввода (один за вызов). |
| `IsKeyDown(Keys key)` | Клавиша зажата в текущем кадре. |
| `IsKeyUp(Keys key)` | Клавиша отпущена. |
| `IsKeyPressed(Keys key)` | Клавиша была нажата в этом кадре (переход из Up в Down). |
| `IsKeyReleased(Keys key)` | Клавиша была отпущена. |
| `MouseScreenPosition` | Позиция мыши в экранных координатах. |
| `MouseReferencePosition` | Позиция мыши в эталонных координатах игры. |
| `IsLeftButtonDown` | Левая кнопка зажата. |
| `IsLeftButtonPressed` | Левая кнопка была нажата в этом кадре. |
| `IsLeftButtonReleased` | Левая кнопка была отпущена в этом кадре. |

## Системные компоненты

### SpriteRenderer

Отвечает за отрисовку спрайта.

| Свойство | Описание |
|----------|----------|
| `TexturePath` | Путь к текстуре относительно папки Content. |
| `Texture` | Загруженная текстура (заполняется автоматически). |
| `Color` | Цветовой оттенок. |
| `SourceRectangle` | Область для отрисовки (`null` – вся). |
| `Effects` | `SpriteEffects` (отражение). |
| `LayerDepth` | Глубина для сортировки (0.0 – передний план, 1.0 – дальний). |
| `SortingLayer` | Целочисленный слой; чем больше, тем позже рисуется (выше). |

### UIText

Отображает текст с помощью `SpriteFont`. Поддерживает горизонтальное и вертикальное выравнивание.

| Свойство | Описание |
|----------|----------|
| `FontPath` | Путь к `.spritefont` в Content. |
| `Font` | Загруженный шрифт. |
| `Text` | Отображаемая строка. |
| `Color` | Цвет текста. |
| `Scale` | Масштаб текста. |
| `Alignment` | Горизонтальное выравнивание: `Left`, `Center`, `Right`. |
| `VerticalAlignment` | Вертикальное выравнивание: `Top`, `Center`, `Bottom`. |
| `LayerDepth`, `SortingLayer` | Управление порядком отрисовки. |
| `GetTextLocalPositionAtIndex(int index)` | Возвращает локальную позицию левого края символа с указанным индексом. Используется `UIInputField` для курсора. |

### UIButton

Интерактивная кнопка, реагирующая на мышь. Использует `InputManager`.

| Свойство | Описание |
|----------|----------|
| `Interactable` | Включена ли кнопка (по умолчанию `true`). |

**События:**
- `OnFocusEnter` / `OnFocusExit` – курсор вошёл / вышел за границы.
- `OnPointerDown` / `OnPointerUp` – нажатие / отпускание левой кнопки над кнопкой.
- `OnClick` – клик (нажатие и отпускание внутри кнопки).
- `OnPressed` – вызывается каждый кадр, пока кнопка нажата и курсор над ней.

### Animation

Компонент для анимации значений (float, Vector2, Color и т.д.) с поддержкой клипов и дорожек. Описание полностью соответствует версии 2.1 (см. исходный файл Animation.cs).

### UIInputField

Текстовое поле ввода с мигающим курсором, поддержкой русского языка и автоповтором. **Полностью полагается на `InputManager`** (включая символы через `TextInput`).

| Свойство | Описание |
|----------|----------|
| `Text` | Текущий текст. Изменение вызывает `OnTextChanged`. |
| `MaxLength` | Максимальная длина (по умолчанию 20). |
| `Placeholder` | Текст-подсказка, когда поле пустое и не в фокусе. |
| `ReadOnly` | Запрещает ввод. |
| `IsFocused` | Находится ли поле в фокусе. Устанавливается кликом по полю. |

**События:**
- `OnTextChanged(string newText)` – вызывается при любом изменении текста.
- `OnSubmit(string text)` – при нажатии Enter (после автоматической очистки поля).

**Методы:**
- `Clear()` – очищает текст и сбрасывает курсор.

**Особенности:**
- Автоматически создаёт дочерний GameObject-курсор с текстурой `Sprites/Panel`.
- Для позиционирования курсора использует `UIText.GetTextLocalPositionAtIndex()`; поэтому на объекте должен присутствовать `UIText`.
- Для ввода символов используется `InputManager.TryGetTextInput()` – в него попадают символы, приходящие через системное событие `Window.TextInput` (на которое подписывается `Game1`).

## Игровые механики

### Роботы и враги

**Robot** – игровой юнит. Хранит здоровье, слоты навыков (`SkillSlots`) и привязку букв к действиям (`LetterBindings`).

```csharp
var robot = new Robot("Альфа", maxHealth: 100);
robot.SkillSlots.Add(new AttackAction());
robot.LetterBindings['а'] = robot.SkillSlots[0];
```

Основные методы:
- `Heal(int amount)`, `TakeDamage(int damage)` – изменение здоровья.
- `ActOnLetter(char letter, BattleContext context)` – выполняет привязанное действие на указанную букву.

**Enemy** – противник. Имеет здоровье и словарь действий на буквы (`Actions`). Действие врага – экземпляр `EnemyAction`.

```csharp
var enemy = new Enemy("Волк", maxHealth: 60);
enemy.Actions['а'] = new EnemyAction { Damage = 8, Description = "царапает" };
```

Методы:
- `TakeDamage(int damage)`, `Heal(int amount)`
- `ActOnLetter(char letter, BattleContext context)` – выполняет своё действие.

### Действия (ActionBase)

Все действия роботов наследуются от абстрактного класса `ActionBase`.

```csharp
public abstract class ActionBase
{
    public string Name { get; set; }
    public string Description { get; set; }
    public abstract void Execute(BattleContext context);
}
```

Контекст `BattleContext` содержит ссылки на текущий `BattleManager` и робота-исполнителя (`User`). Готовые действия: `AttackAction`, `HealAction`. Чтобы добавить новое (например, «Анализ»), достаточно создать класс-наследник и реализовать `Execute`.

### Менеджер битвы (BattleManager)

Компонент, управляющий боем. Крепится к корневому объекту `BattleRoot`. При активации:
- Инициализирует врагов, копирует список роботов из `GameState.Robots`.
- Подписывается на `UIInputField.OnSubmit`.
- При получении слова проверяет его реальность через `WordValidator.IsRealWord`, затем выполняет посимвольную обработку.

**Важные методы и события:**
- `OnLogMessage(string msg)` – событие для обновления лога в UI.
- `OnWordAccepted(string word)` – вызывается перед выполнением слова.
- `OnBattleEnd(bool victory)` – вызывается при завершении боя.
- `LogMessage(string msg)` – добавляет сообщение во внутренний буфер и вызывает событие `OnLogMessage`.
- `GetRandomAliveRobot()`, `GetRandomAliveEnemy()` – вспомогательные методы для выбора цели.

**Поток обработки слова:**
1. `OnWordSubmitted(word)` → проверка `WordValidator.IsRealWord`. Если слово не распознано, выводится сообщение и ввод не происходит.
2. Проверка спецэффектов (если слово есть в `special_words.csv`).
3. Посимвольное выполнение:
   - Для каждой буквы вызываются `ActOnLetter` у живых роботов и врагов.
4. Проверка условий победы/поражения.

### Валидатор слов (WordValidator)

Статический класс, отвечающий за проверку слов.

**Инициализация:**
- `WordValidator.Initialize()` – загружает слова из `Content/words.txt` (по одному слову на строку) и спецслова из `Content/special_words.csv` (формат: `слово,эффект`).

**Если файл отсутствует**: `IsRealWord` всегда возвращает `true` (проверка отключена). Таким образом, игра не ломается.

**Методы:**
- `IsRealWord(string word)` – возвращает `true`, если слово есть в словаре или словарь пуст.
- `GetSpecialEffect(string word)` – возвращает строку-идентификатор спецэффекта или `null`.

### Глобальное состояние игры (GameState)

Хранит данные, сохраняющиеся между боями: список роботов (`Robots`) и множество использованных слов (`UsedWords`).

```csharp
GameState.Reset();                      // очистка
GameState.Robots.Add(new Robot(...));
GameState.UsedWords.Contains("слово");
```

### Менеджер настройки протокола (ProtocolSetupManager)

Компонент, управляющий привязкой букв к действиям на экране настройки. Не управляет UI напрямую – только предоставляет методы, вызываемые UI.

- `BindAction(char letter, ActionBase action)` – назначает действие на букву текущему роботу.
- `UnbindAction(char letter)` – удаляет привязку.
- `NextRobot()` / `PreviousRobot()` – переключение текущего робота.
- `CurrentRobot` – возвращает текущего робота или `null`.

Переключение робота и обновление UI выполняется через `MySceneBuilder.RefreshProtocolUI()`.

## Построение сцен и интерфейса

### Класс MySceneBuilder

Статический класс, в котором декларативно описывается весь интерфейс и создаются менеджеры. Главная точка входа – метод `Build()`.

```csharp
public static void Build()
{
    Scene.Initialize();
    // инициализация GameState, WordValidator
    CreateMainMenu();
    CreateBattleMode();
    CreateProtocolSetupMode();
    SwitchToMainMenu();
}
```

**Переключение режимов:**
- `SwitchToMainMenu()` – главное меню.
- `SwitchToBattle()` – бой (очищает поле ввода).
- `SwitchToProtocolSetup()` – настройка протокола (вызывает `RefreshProtocolUI()`).

Каждый режим имеет свой корневой GameObject, который активируется/деактивируется.

### Главное меню

Содержит кнопки: «Играть», «Настройка протокола» (обе ведут на экран протокола), «О проекте», «Выход». Создаётся один раз.

### Режим битвы

- Поле ввода `UIInputField` в центре экрана.
- Лог битвы (`UIText`) в левом верхнем углу.
- Компонент `BattleManager` прикреплён к корню боя.
- Кнопки: «В меню» (слева снизу), «Протокол» (справа снизу, для тестирования).

### Настройка протокола

Включает:
- **Верхняя панель:** кнопки `<` / `>` для смены робота и отображение имени текущего робота.
- **Сетка букв:** 33 кнопки (русский алфавит), размещённые в сетке. При клике буква выделяется, повторный клик снимает выделение.
- **Панель навыков справа:** динамически создаваемые кнопки для каждого навыка текущего робота. Клик назначает навык на выделенную букву.
- **Кнопка «Удалить привязку»** – снимает привязку с выделенной буквы.
- **Кнопки навигации:** «В бой», «В меню».

`RefreshProtocolUI()` обновляет все подписи и цвета в соответствии с состоянием текущего робота.

## Расширение функционала

### Добавление нового действия робота

1. Создайте класс в папке `Gameplay/Actions/`, унаследованный от `ActionBase`.
2. Реализуйте метод `Execute(BattleContext context)`. Используйте `context.Battle` для доступа к `BattleManager` и `context.User` для определения робота-исполнителя.
3. Зарегистрируйте действие в слотах робота. Это можно сделать:
   - В `MySceneBuilder.Build()` при начальной инициализации.
   - В фазе настройки протокола, добавив кнопку в список навыков (потребуется модификация `RefreshProtocolUI`).

Пример действия «Сканирование», которое показывает здоровье всех врагов:
```csharp
public class ScanAction : ActionBase
{
    public ScanAction() { Name = "Сканирование"; Description = "Показывает здоровье врагов."; }

    public override void Execute(BattleContext context)
    {
        foreach (var enemy in context.Battle.Enemies)
            context.Battle.LogMessage($"{enemy.Name}: {enemy.CurrentHealth} HP");
    }
}
```

### Добавление нового типа врага или вражеского действия

Враг создаётся через конструктор `Enemy`. Его действия на буквы – экземпляры `EnemyAction`. Для сложных поведений можно создать наследника `EnemyAction` или добавить отдельную логику в `BattleManager.InitializeBattle()`.

### Добавление спецэффектов слов

1. Внесите запись в файл `Content/special_words.csv` в формате: `слово,идентификатор_эффекта`.
2. В `BattleManager.OnWordSubmitted` добавьте обработку полученного идентификатора (например, через `switch` или словарь делегатов).
3. Убедитесь, что `WordValidator.GetSpecialEffect()` возвращает идентификатор.

Пример: `огонь,double_attack` – в битве при вводе слова «огонь» все атаки роботов на этот ход наносят двойной урон.

### Создание новых компонентов

- Наследуйте от `Behaviour` или `UIButton`, `UIText`, если нужна своя отрисовка/логика.
- Для UI обязательно работайте с `Transform` для позиционирования и `ResolutionManager` не используйте напрямую – все координаты задавайте в эталонных единицах.
- Для обработки ввода используйте `InputManager`.
- Если нужен доступ к `ContentManager` после загрузки, используйте `Scene.Content`.

### Работа с пользовательским вводом

**Категорически запрещено** использовать `Keyboard.GetState()` или `Mouse.GetState()` в компонентах. Всегда применяйте методы `InputManager`.  
Для обработки текста используйте `InputManager.TryGetTextInput()`. Символы попадают туда автоматически благодаря подписке `Game1.Window.TextInput` на `InputManager.FeedTextInput`.

## Интеграция с MonoGame

Файл `Game1.cs` должен содержать:

```csharp
public class Game1 : Core
{
    protected override void Initialize()
    {
        MySceneBuilder.Build();
        base.Initialize();
        ResolutionManager.Initialize(GraphicsDevice);
    }

    protected override void LoadContent()
    {
        Scene.LoadContent(Content);
        base.LoadContent();
    }

    protected override void Update(GameTime gameTime)
    {
        InputManager.Update();                // строго один раз в начале кадра
        // ... обработка глобальных клавиш (Escape, F11 и т.п.)
        Scene.Update(gameTime);
        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(...);
        SpriteBatch.Begin(...);
        Scene.Draw(SpriteBatch);
        SpriteBatch.End();
        base.Draw(gameTime);
    }
}
```

Обязательно:
- `InputManager.Update()` перед `Scene.Update()`.
- `ResolutionManager.Initialize()` после инициализации графического устройства.
- Подписка `Window.TextInput += (s, args) => InputManager.FeedTextInput(args.Character);` (обычно в конструкторе `Game1`).

## Советы и лучшие практики

1. **Всегда используйте эталонные координаты (1920×1080)** при задании `Position` и `Size`.
2. Для адаптивных интерфейсов применяйте `SizeMode.Stretch` с отступами.
3. Не изменяйте коллекции сцены во время итерации – `Scene.Update` и `Scene.Draw` это учитывают.
4. Кэшируйте ссылки на компоненты в `Start()`, чтобы избежать вызова `GetComponent<T>()` каждый кадр.
5. Для временного отключения объектов используйте `ActiveSelf` вместо уничтожения.
6. При работе со словарём: файл `words.txt` должен быть в кодировке UTF-8 (без BOM), каждое слово с новой строки.
7. Для отладки отключайте вывод в консоль, либо оборачивайте в `#if DEBUG … #endif`.
8. Новые действия создавайте как можно более общими – используйте параметры в `BattleContext` или наследниках `ActionBase`, чтобы избежать жёстких привязок к конкретным врагам.

## Правила оформления кода

- **Все публичные члены** должны иметь XML-комментарии `<summary>`.
- Комментарии отвечают на вопрос «почему?», а не «что?».
- **Именование:** классы и методы – `PascalCase`, приватные поля – `_camelCase`, свойства – `PascalCase`, локальные переменные – `camelCase`.
- **Организация файлов:** движок в `Engine/`, игровая логика в `Gameplay/`, строитель сцены в `Scenes/`.
- **Структура класса:** поля → свойства → конструкторы → публичные методы → приватные методы.
